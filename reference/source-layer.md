# 圆酱的营养师膳食评价助手skill｜资料调用层级

本文件规定《圆酱的营养师膳食评价助手skill》如何调用既有营养资料、指南和蒸馏材料。

核心原则：**客户记录是事实层，指南/资料是解释层，营养师判断是决策层。三层不能混在一起。**

---

## 1. 可调用资料类型

### L1｜客户原始记录（最高优先级事实层）

包括三日饮食文字、客户聊天打卡、表格饮食记录、语音转文字、图片/照片经人工或模型描述后的文字，以及营养师补充的客户背景。

使用规则：

- 只把客户确实说过/表格确实写过的内容作为事实。
- 没说就是“未记录/待确认”，不是“没有吃”。
- 原话保留，结构化理解另列。

### L2｜中国居民膳食指南等通用指南（通用原则层）

可使用的资料示例：

- 《中国居民膳食指南（2022）》等公开发布的通用膳食指南。
- 使用者/营养团队已合法取得并确认可用于本项目的指南、教材、手册或团队内部资料。

公开版不绑定任何本地文件路径；实际部署时由营养师团队自行配置资料来源，并确认版权与使用权限。

可用于：

- 判断饮食结构是否均衡。
- 提供“增加蔬菜、全谷物、优质蛋白、少油盐糖酒”等通用方向。
- 解释为什么要规律餐次、减少含糖饮料、提升食物多样性。

不可用于：

- 对单个客户做疾病诊断。
- 替代个体化处方。
- 把三日粗略记录直接换算成精确膳食宝塔达标率。

### L2B｜DRIs 2023 与食物成分数据（计算参考层）

用于“摄入量估算与参考摄入量差距”时，先读：`reference/nutrient-calculation-design.md`。

- DRIs 2023：只作为参考摄入量对照；必须来自已核验原表/页码，不能用搜索摘要或其他国家 RDA 顶替。
- 食物成分数据：优先中国食物成分表、包装营养成分表、品牌公开数据或营养师确认数据；经验估算必须标注低/中置信度。
- 输出差值时必须同时写 `摄入量估算来源`、`DRIs 来源/表页`、`差值/达标比例`、`置信度`。
- 三日记录低于 RNI/AI 只能说“可能偏低/需要补问”，不能诊断缺乏症；超过 UL 或涉及补充剂/特殊人群时转人工。

### L3｜项目已蒸馏营养资料（证据/背景层）

主要目录：

- `research/营养学调研/distillations/`
- ledger：`research/营养学调研/distillations/distillation_ledger.jsonl`
- 重建脚本：`research/营养学调研/scripts/rebuild_distillation_ledger.py`

使用规则：

- 优先使用 ledger 中 `distilled`、且正文能定位原文的材料。
- `metadata_or_landing_report`、`source_mismatch_report`、`retracted_or_unusable_report` 不能作为强结论来源。
- 文献证据用于背景解释，不直接替代膳食记录分析。
- 如果要引用具体研究结论，必须回到对应报告核对 DOI/PMID/PMCID、研究对象、干预、结局和边界。

适合用在：解释营养问题的一般背景、内部总结证据边界、提醒“这个结论不能外推”。不适合直接给客户堆论文或由单个研究推出个体处方。

### L4｜专病食养 skill（专病事实层）

如果客户记录中出现明确专病/特殊人群，应调用对应专病 skill，而不是只用本技能。

已知示例：

- `child-obesity-food-guide`：儿童青少年肥胖。
- 未来可能有 adult-obesity / diabetes / hypertension / hyperuricemia 等。

本技能负责三日记录结构化和模式分析；专病 skill 负责疾病相关边界、红线、指南事实。

## 2. 输出时如何标注资料来源

默认不需要给客户端长篇引用，但内部总结应区分：

```text
【来自客户记录】连续两天早餐未记录蛋白质来源。
【来自通用指南原则】早餐可优先补充蛋/奶/豆制品等优质蛋白来源，并搭配主食和蔬果。
【需要补问】早餐是否实际喝了牛奶/豆浆未记录，需确认。
```

推荐内部标记：

- `记录事实`：来自客户原话/表格。
- `结构判断`：由三日记录推断的饮食模式。
- `指南原则`：来自中国居民膳食指南等通用指南。
- `证据背景`：来自已蒸馏文献或专题资料。
- `待确认`：信息不足。
- `转人工`：超出本技能边界。

## 3. 调用资料的操作流程

### 只是整理三日记录

1. 先完成结构化表。
2. 标注缺失信息和置信度。
3. 用通用指南原则给 3–5 条改善建议。
4. 不强行翻文献。

### 要求“更专业/有依据/结合指南”

1. 先完成结构化表。
2. 读取/检索通用指南或已蒸馏资料。
3. 只把资料用于解释“为什么这样建议”。
4. 输出中标注“通用原则，不等于个体化处方”。

### 涉及疾病、药物、儿童、孕产、老人、慢病

1. 先做红线判断。
2. 调用对应专病 skill 或转人工。
3. 本技能只保留饮食记录整理，不给疾病处方。

## 4. 不确定性表达模板

- “这条判断来自三日记录的模式观察，不是精确称量计算。”
- “由于份量没有记录，热量只能判断趋势，不能当作准确摄入。”
- “这个建议属于通用膳食结构优化，不替代针对疾病/用药的个体化方案。”
- “如果要判断是否适合你本人，需要结合身高体重、目标、活动量和健康状况。”
- “这部分已经超出普通饮食记录分析，建议转给专业医师或临床营养师确认。”

---

## 5. 三日记录分析中最常用的通用原则

在没有特殊疾病和禁忌时，可作为通用方向：

1. **规律餐次**：先修复漏餐、夜宵、长时间空腹后的暴食。
2. **每餐有蛋白质来源**：蛋、奶、豆制品、鱼禽肉等，按客户偏好选择。
3. **蔬菜优先补足**：尤其外卖/聚餐日，先加一份蔬菜。
4. **主食不极端**：减少精制主食单一化，逐步加入全谷、杂豆、薯类。
5. **控制隐性能量**：奶茶、甜饮、酒精、坚果、零食、汤汁拌饭。
6. **外卖先做小替换**：少汤汁、加蔬菜、少炸物、主食适量。
7. **记录质量也是干预点**：先让客户学会记录份量和场景，而不是一上来要求完美饮食。

---

## 6. 禁止做法

- 禁止把文献蒸馏结论直接变成客户剂量建议。
- 禁止用指南压客户：“你必须达到……”
- 禁止输出没有来源的精确克数/千卡。
- 禁止在客户记录不足时评判人格、自律、意志力。
- 禁止把三天记录外推为长期饮食习惯，除非客户明确说“长期如此”。
