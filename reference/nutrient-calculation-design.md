# 营养素摄入量计算与 DRIs 2023 对照设计

本文件规定 `three-day-diet-record-analyzer` 如何从三日饮食记录进入“可解释、可追溯、有置信度”的营养素摄入估算，并与《中国居民膳食营养素参考摄入量（2023版）》（DRIs 2023）进行差距输出。

> 重要边界：DRIs 参考摄入量用于膳食设计、群体/个体膳食评估与风险提示，不等于疾病诊断阈值；三日饮食记录通常只能反映短期摄入模式，不能直接诊断“缺乏症”或给出治疗剂量。

---

## 1. 计算链路总览

按以下顺序执行，不得跳步：

```text
客户原始记录
  ↓
三日 × 餐次结构化
  ↓
食物标准化与份量估计
  ↓
调用食物成分数据估算摄入量
  ↓
按年龄/性别/孕哺/生理状态匹配 DRIs 2023 参考值
  ↓
计算差值、达标比例、状态、置信度
  ↓
输出：客户友好版 + 营养师内部版
```

### 必须保留的三类来源

1. **记录来源**：客户原话、表格、语音转写、图片描述。
2. **食物成分来源**：中国食物成分表、包装营养成分表、餐饮品牌公开营养信息、营养师手工录入数据等。
3. **参考摄入来源**：DRIs 2023 原表或经人工校对后的内置表。

任何一个环节缺来源，都要把对应结果降级为“估算/不可判断”。

## 2. 输入所需人群信息

要做 DRIs 对照，至少需要：

| 字段 | 是否必需 | 说明 |
|---|---:|---|
| 年龄 | 必需 | 用于匹配 DRIs 年龄组。没有年龄时只能输出摄入量估算，不做参考值差距。 |
| 性别 | 多数必需 | 多数年龄段参考值按男女分层；儿童早期部分可能不分性别。 |
| 孕期/哺乳期 | 条件必需 | 女性如处于孕期/哺乳期必须单独匹配，不得用普通成人值代替。 |
| 身高体重 | 建议 | 用于解释能量、蛋白质、体重管理目标；不是所有 DRIs 项都需要。 |
| 活动水平 PAL | 建议 | 能量需要量强依赖活动水平。缺失时只能做粗略能量趋势。 |
| 疾病/用药/特殊状态 | 安全必需 | 肾病、糖尿病、痛风、孕产、儿童、老人、进食障碍等需转人工或调用专病 skill。 |

缺少年龄/性别/孕哺状态时，输出中使用：`参考值匹配：不可判断（缺少必要人群信息）`。

## 3. 食物摄入估算字段

推荐使用 `assets/food-intake-estimate-template.csv` 的字段。核心字段如下：

| 字段 | 含义 |
|---|---|
| day / meal | 第几天、餐次。 |
| raw_description | 客户原始描述。 |
| normalized_food | 标准化食物名，如“米饭”“煎鸡蛋”“黄焖鸡外卖”。 |
| amount_g | 估计可食部克数；无法估计时留空。 |
| amount_basis | 份量依据，如“客户给出克数”“一拳米饭按约100g估算”“包装标示”。 |
| amount_confidence | high / medium / low / unknown。 |
| food_composition_source | 食物成分来源，如“中国食物成分表”“包装营养成分表”“品牌公开数据”“经验估算”。 |
| energy_kcal, protein_g, fat_g, carbohydrate_g, fiber_g | 常用能量与宏量营养素。 |
| vitamin_* / mineral_* | 维生素和矿物质字段，按可得数据填写。 |
| assumptions | 关键假设。 |

### 份量置信度

| 置信度 | 条件 | 可做什么 |
|---|---|---|
| high | 有称重、克数、包装营养成分表、明确品牌规格，或营养师确认。 | 可输出数值摄入量与 DRIs 差距，但仍标注来源。 |
| medium | 有碗/拳/掌/个数/常见外卖份量，可用常见换算。 | 可输出区间或粗估数值，差距状态用“可能”。 |
| low | 只有“一些/一份/正常/聚餐/随便吃点”。 | 不做精确差距，只做模式判断和补问信息。 |
| unknown | 食物或份量均不清楚。 | 标记不可估算。 |

## 4. DRIs 2023 内置表字段

推荐使用 `assets/dri2023-core-template.csv` 的字段维护内置参考值。**只有从原书/PDF/权威表格核对后的数值才能填入。**

| 字段 | 含义 |
|---|---|
| nutrient_id | 机器可读 ID，如 `protein`, `vitamin_c`, `calcium`, `iron`。 |
| nutrient_name_zh | 中文名。 |
| nutrient_group | `energy` / `macronutrient` / `vitamin` / `mineral` / `other_dietary_component`。 |
| unit | 单位，如 `kcal/d`, `g/d`, `mg/d`, `μgRAE/d`, `%E`。 |
| dri_type | EAR / RNI / AI / UL / AMDR / PI-NCD / SPL 等。 |
| age_min / age_max | 年龄范围；用年表示，不足 1 岁可用小数或月龄字段扩展。 |
| sex | female / male / all。 |
| physiological_status | general / pregnancy_T1 / pregnancy_T2 / pregnancy_T3 / lactation_0_6m / lactation_7_12m 等。 |
| value | 参考值。 |
| source_title | 来源标题，默认《中国居民膳食营养素参考摄入量（2023版）》。 |
| source_page_or_table | 页码/表号；必须可追溯。 |
| source_status | verified / pending / conflict / deprecated。 |
| note | 单位换算、适用边界、与 2013 版差异等。 |

## 5. 对照计算公式

### 5.1 单一参考值（RNI/AI/EAR 等）

```text
intake = 三日平均每日摄入量
reference = 匹配人群的 DRIs 参考值
absolute_difference = intake - reference
percent_of_reference = intake / reference × 100%
```

推荐状态：

| 状态 | 建议判定 |
|---|---|
| 不可判断 | 缺少年龄/性别/孕哺状态、份量置信度低、无对应 DRIs 值、食物成分缺失。 |
| 可能偏低 | intake 明显低于 RNI/AI，且估算置信度中高；措辞用“可能”，不说“缺乏”。 |
| 接近参考 | intake 接近参考值；注意三日记录代表性有限。 |
| 达到/大致达到 | intake 达到 RNI/AI 或处于合理范围。 |
| 可能偏高 | intake 明显高于建议范围，尤其是钠、添加糖、脂肪供能比等。 |
| 超过 UL 风险 | intake 超过 UL；优先提示风险与转人工，尤其来自补充剂时。 |

### 5.2 范围型参考值（AMDR、脂肪/碳水供能比等）

```text
energy_percent = nutrient_energy_kcal / total_energy_kcal × 100%
```

与下限/上限比较：低于范围、范围内、高于范围。若总能量估计低置信，则供能比也降级。

### 5.3 UL 上限

UL 只用于安全上限风险提示。不要把“低于 UL”说成“推荐吃到 UL”。超过 UL 时：

1. 标注具体来源：食物、强化食品、补充剂、药物/保健品。
2. 建议停止自行加量并转专业人员确认。
3. 不给治疗性调整剂量。

### 客户端措辞模板

- “这部分是基于你记录的食物和份量做的估算，不是称重膳食调查。”

## 8. 不允许的做法

- 不允许在没有 DRIs 2023 原表核验的情况下填入看似精确的参考值。
- 不允许把 2013 版、美国 RDA、WHO/FAO 或其他国家标准混写成“中国 DRIs 2023”。
- 不允许把搜索摘要、百科、营销号、二次转载页作为唯一来源写入内置参考值。
- 不允许把三日摄入低于 RNI/AI 直接说成“你缺乏某营养素”。
- 不允许在客户记录份量不清时输出一长串精确到个位数的营养素差值。
- 不允许根据本 skill 给补充剂治疗剂量；补充剂、孕产、儿童、慢病、用药必须转人工/专业人员。

## 9. 外部 skill 参考与融合边界

本项目曾参考 GitHub/网页上非本项目的营养 skill 结构，详见 `reference/external-skill-synthesis.md`。

可借鉴点：

- `nutrition-analyzer`：趋势分析、达标率、风险识别、运动/睡眠/慢病相关性分析的结构。
- `health-coach`：用户画像、目标、长期日志、份量估计与中国常见食物参照的组织方式。
- `Nutrition-Macro-Calculator`：宏量目标、餐单规划、饮食限制支持的任务边界。
- `SkillsBot 营养专家`：先检查/建立偏好、过敏原、饮食限制、目标，再做个性化建议。
- `nutrition-infographic-skill`：可视化交付前先明确用途、受众、渠道、风格和可画数据。

不可照搬点：

- 不默认使用美国 RDA 或 USDA 数据替代中国 DRIs/中国食物成分表。
- 不照搬其示例中的营养素数值、达标阈值或补充剂建议。
- 不因外部 skill 有“600+ 食物库/健身公式/个性化食谱”字样，就绕过本项目的来源分级和人工校对。
