# 食物成分表查表与一餐营养素换算规则

主用数据文件：`assets/food-composition-data/china_food_composition_pdf_section_a_validated_20260528.csv`。  
辅助规范化文件：`assets/food-composition-data/cn_food_composition_1991_v0.jsonl`（含 `name_base` / `name_note` / `data_version_hint` 等字段）。

## 数据源版本口径

使用本表时必须写明：数据来自圆酱提供的腾讯文档 PDF《中国食物成分表》，Section A 经抽样原文对照；全网交叉验证高度推断为 **1991 版《中国食物成分表》子集/简表抄录**，不是 2018 标准版第 6 版，也不是现行官方完整数据库。适合 demo / B-limited 查表，不适合作临床唯一依据。

## 字段覆盖

可计算：能量、水分、蛋白质、脂肪、膳食纤维、碳水化合物、维生素A、维生素B1、维生素B2、烟酸、维生素E、钠、钙、铁、维生素C、胆固醇。

不可声称已覆盖：钾、磷、镁、锌、硒、铜、锰、碘、氟等多数常量/微量元素。

## 一餐计算流程

1. **拆食物项**：把一餐拆成单独食物或菜肴组成，例如“白粥100g + 鸡蛋1个 + 榨菜10g”。
2. **确认重量口径**：优先使用“可食部克数”。若是带骨/带皮/带壳/包装总重，先按可食部比例修正，并记录假设。
3. **匹配食物名**：
   - 先 exact match `food_name` / `name_raw`；
   - 再用 `name_base` 和括号内别名做人工辅助匹配；
   - 不确定时列为“待人工匹配”，不要硬套相近食物。
4. **逐项换算**：对每个已匹配食物，用下方公式计算各营养素。
5. **每餐小计**：同一餐各食物相加，输出能量、三大宏量、膳食纤维，以及本数据源覆盖的维生素/矿物质。
6. **全天/三日合计**：在餐次小计基础上再汇总；不要把未记录餐次当作“没吃”。
7. **列出未覆盖项**：凡用户要求钾、磷、镁、锌、硒等，本数据源应输出“未覆盖，需补完整第6版或其他合规数据源”。

## 换算公式

对每个食物项：

```text
营养素摄入量 = 食物成分表每 100g 可食部数值 × 估计可食部克数 / 100
```

例如某食物蛋白质 13.6 g/100g，可食部摄入 50g，则蛋白质约 `13.6 × 50 / 100 = 6.8g`。

## 输出要求

- 每个数值必须带置信度：高 / 中 / 低。
- 每餐小计、全天合计、三日均值要分开。
- 必须列“数据源与版本”：1991 版谱系子集/腾讯文档 PDF/Section A 抽取/非第6版。
- 未匹配食物、重量不明、烹调损失不明、品牌包装差异必须列入“不确定项”。
- 不同版本食物成分表不得混算；如果未来接入第6版，应显式切换数据源。
